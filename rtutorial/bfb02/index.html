Beginner for Beginners 02: データと変数作成

May 20, 2021
beginnerforbeginners

Introduction

　前回では、パッケージの準備、データの読み込み、変数名の変更を行いました。本回では、引き続き、四則演算、変数の作成、合併について取り上げます。ここでは、すぐに使える、かつ私がいつも使っている手法を中心に紹介している予定ですが、ここで紹介されたもの以外にも、多様な手法があります。基本的にはtidyverseのパッケージによる処理方法をご紹介します。

データの準備

　では、前回で用意できたプロジェクトを開けて、libraryを読み込んでください。

1

library("tidyverse")

　プロジェクトには、jgss2012のデータが確実に入ってあることを確認しましょう。

1

view(jgss2012)

以上のコードを走らせることで、データセットの新しいウィンドが自動的に開かれ、データの中身について確認できます。なんとなくデータがちゃんと入ってあることを確認できたら、スクリプトに戻って、今回の内容をやってみましょう。
または、R Studioの右にあるデータ欄から、jgss2012をクリックして、データセットを確認することもできます。

四則演算

Rでは、もちろん四则运算ができます。数値による四则运算はもちろん、変数間の四则运算もできます。四则运算によって、新しい合成変数などを作成することができます。
例えば、dobyearという出生年度から年齢を算出したい場合は、

1

jgss2012$age <- 2011-jgss2012$dobyear

と打ちます。前回で説明したように、<-は付与する意味で、調査を行なった年2011から出生年度を引いた結果を、左のageという変数に書き込むということを指します。
演算したら、データがきちんと入っているかをチェックしましょう。

1
2

jgss2012$age %>% 
  str()

num [1:4667] 87 86 84 82 82 82 81 81 80 80 …

1
2

jgss2012$age %>% 
  head()

[1] 87 86 84 82 82 82

1
2

jgss2012$age %>% 
  table()

以上の三つの方法が少なくともデータがきちんと入っているかを確認することができる。以上のいずれかで確認したら良いと思います。

変数の作成

これから実際の分析を行う前には、欠損値などの前処理をする必要ができています。JGSSのような社会調査データでは、基本的に欠損値や「わからない」の答えを「999」などの数値で入力されるケースがほとんどである。この場合には、コードブックを参照しながら、データの前処理を行いましょう。

欠損値の処理

欠損値の処理には、多くの方法がありますが、ここでdyplrを使用する方法と、一般的な方法(?)の二種類を紹介します。
まず、標準関数を使用すると、以下のコードを使用します。

1
2

jgss2012$xjobhwk[jgss2012$xjobhwk == 888 
　　　　　　　　　| jgss2012$xjobhwk == 999] <- NA　# |はあるいはの意味

ここでは、888→わからない、999→無回答の二つの欠損値があるので、その二者を同時に処理すると、あるいはの意味で、|を入れます。このコードは、jgss2012のデータセットの中のxjobhwkという変数のなかの、888と999を欠損値NAとして処理してください、ということです。

また、dyplrを使用する場合、na_ifの関数を使用します。

1
2
3
4
5

jgss2012 <- jgss2012 %>% 
  mutate(xliveyr = na_if(xliveyr, 99))

jgss2012$xliveyr %>% 
  table()

| 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 |
| :—-: | :—-: | | :—-: | :—-: | | :—-: | :—-: | | :—-: | :—-: | |774 |111 | 253 | 204 | 454 | 800| 668| 1393|

ここでは、99という欠損値が存在する変数xliveyrを、mutate関数で処理した。意味としては、jgss2012のデータセットの中のxliveyrという変数のなかの、全ての99を欠損値として変換してください。という意味です。

変数の変換

以上では、欠損値の処理方法について紹介しました。これからは、変数の変換とりわけ値の再付与について取り上げます。
JGSS2012の性別を例にします。性別では１→男性、２→女性として、データセットに入っていることをまず確認してみましょう。

1
2

jgss2012$sexa %>% 
  table()
1 2　
2148 2519

このような２値を、１と０に男性ダミー変数を作ると、以下のようなコードを書きます。

1
2
3

jgss2012 <- jgss2012 %>% 
  mutate(male_dummy = case_when(sexa == 2 ~ 0,
                                sexa == 1 ~ 1))

ここでも、dyplrパッケージの中のmutate関数を使用しています。今回は、na_ifという欠損値を処理する関数ではなく、case_whenという条件文を使用します。
ここでは、JGSS2012のデータセットのにあらためmale_dummyという変数を作って、sexaの変数の中の1を1のままとして、2を0に変換する意味です。また、この変換は、文字型でも使用できます。

1
2
3
4
5

jgss2012 <- jgss2012 %>% 
  mutate(gender = case_when(sexa == 1 ~ "男",
                            sexa == 2 ~ "女"))
jgss2012$gender %>% 
  table()
女 男
2519 2148

変数の作成

それから、データセットの中から、条件に相応しい項目だけを抽出して新しい変数を生成したりすることができます。
例えば、男性で、65歳以上のサンプルを抽出して、OBとして新しいダミー変数を作成したい場合は、以下のコマンドでできます。

1
2
3
4

jgss2012$ob <- NA
jgss2012$ob[jgss2012$male_dummy == 1 & jgss2012$ageb > 65] <- 1
jgss2012$ob %>% 
  table()

まず、OBという変数をあらかじめ作り、男性ダミーが１と年齢が65以上の条件にあうサンプルだけに１を付与する意味です。　　

それから、サンプルの中で、どれかに合えば良いケースもあるでしょう。その場合には、前節で取り上げた|を使用します。
例えば、ここで、学歴について、初等教育、中等教育、高等教育について分けたい場合は、以下のように、それぞれについて付与します。

1
2
3
4
5
6
7
8

jgss2012$educ <- NA
jgss2012$educ[jgss2012$xxlstsch == 1|jgss2012$xxlstsch == 2|
                jgss2012$xxlstsch == 3|jgss2012$xxlstsch == 8] <- "primary"
jgss2012$educ[jgss2012$xxlstsch == 4|jgss2012$xxlstsch == 5|
                jgss2012$xxlstsch == 6|jgss2012$xxlstsch == 9|
                jgss2012$xxlstsch == 10|jgss2012$xxlstsch == 11] <- "secondary"
jgss2012$educ[jgss2012$xxlstsch == 7|jgss2012$xxlstsch == 12| 　　　　　　　　　　jgss2012$xxlstsch ==13] <- "tertiary"

また、もっと簡潔な方法について、slackをご参照ください（次回追加）。
