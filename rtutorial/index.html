Beginner for Beginners 03: データ作成と合成変数

Introduction

　今日は、前回残っていたファクター変数の作成のもう一つの手法について説明した上、合成変数の作成と信頼度について取り上げます。実際には、因子分析などを経て、合成変数を作成することは普通ではありますが、とりあえず、データの作成の部分で合成変数の作成の仕方、信頼性の判断の仕方について書きます。

データの作成

前回で、|を使ったデータの作成の仕方について、学歴を例として以下のように書きました。

1
2
3
4
5
6
7
8
9

jgss2012$educ <- NA
jgss2012$educ[jgss2012$xxlstsch == 1|jgss2012$xxlstsch == 2|
                jgss2012$xxlstsch == 3|jgss2012$xxlstsch == 8] <- "primary"
jgss2012$educ[jgss2012$xxlstsch == 4|jgss2012$xxlstsch == 5|
                jgss2012$xxlstsch == 6|jgss2012$xxlstsch == 9|
                jgss2012$xxlstsch == 10|jgss2012$xxlstsch == 11] <- "secondary"
jgss2012$educ[jgss2012$xxlstsch == 7|jgss2012$xxlstsch == 12|  
                jgss2012$xxlstsch ==13] <- "tertiary"

これでも、そのまま使えますが、何回も変数名のコピペをしないといけないという煩雑な作業があります。そこで、もっと簡単にやりたい場合には、%in%を使用します。この演算子は、簡単にいうと「含まれる」という意味です。

そこで、この演算子を使って、新しい変数を作成すると、こうなります。

1
2
3
4
5

jgss2012$educ <- jgss2012$xxlstsch #そのままxxlstschのデータをeducという変数にコピーする
jgss2012$educ[jgss2012$educ %in% c(1,2,3,8)] <- "primary"
jgss2012$educ[jgss2012$educ %in% c(4,5,6,9,10,11)] <- "secondary"
jgss2012$educ[jgss2012$educ %in% c(7,12,13)] <- "tertiary"
jgss2012$educ[jgss2012$educ %in% c(14, 99)] <- NA

このように、学歴が%in%の後の数値のどれかと一致する場合、←の後の文字列で入れ替わるという意味です。

また、それぞれの数値を事前に定義してから、%in%を用いることもできます。

1
2
3
4
5
6
7
8

primary <- c(1,2,3,8)
secondary <- c(4,5,6,9,10,11)
tertiary <- c(7,12,13)
na <- c(14, 99)
jgss2012$educ[jgss2012$educ %in% primary] <- "primary"
jgss2012$educ[jgss2012$educ %in% secondary] <- "secondary"
jgss2012$educ[jgss2012$educ %in% tertiary] <- "tertiary"
jgss2012$educ[jgss2012$educ %in% na] <- NA

このようなやり方でもできます。実行したら、データの確認をしましょう。

1

jgss2012$educ %>% 
  table()

間違っていなければ、以下のようなテーブルが返ってくるはずです。

809 2752 1085
以上、%in%の使い方でした（slack参照）。

合成変数と信頼性

JGSSのような社会調査を行う場合、特定の一つの測定したい尺度に、複数の質問項目を設けることがある。このような複数の項目を分析する際に、一つの尺度に統合してからの方が分析しやすく、結果の解釈もしやすくなるかと思います。ここでは、このような変数を合成変数としています。基本的に合成変数を作る際には、質問項目の結果の和を項目数で割リます。

ここで考えないといけないことは、このいくつかの質問項目は、本当に同じ尺度について聞いているのかという項目の内部統合性の問題があります。そこで、Cronbach α で判断します。このαは0から1までの値をとり、値が大きければ大きいほど、データ内部の統合性が良いという意味です。これに関する具体的な説明を省きますが、出し方と判断の仕方について説明します。

ではまず、データの欠損値などの下処理や、逆転項目の逆転を行います。データの処理などには、tidyverseのmutateなどを使ってご自身で行ってみてください。

では、まずCronbach αを算出するために、psychのパッケージを読み取ってください。まだ入れていないのであれば、インストールしてください。

1

library("psych")

今回は、例えば、sfmhpeac, sfmhengy, sfmhdprsの三つの変数を使って、合成変数を作成したいとしましょう。データの下処理が完了しましたら、この三つのデータが含まれるサブセットを作ります。

1
2
3
4
5
6

#やり方１  
a1 <- c("sfmhpeac", "sfmhengy", "sfmhdprs")
a11 <- jgss2012[a1]　　

#やり方２
a11 <- subset(jgss2012, select=c(sfmhpeac, sfmhengy, sfmhdprs))

作成できたら、右のデータの部分に、a11という三つの変数があるデータセットが作成されているはずです。

サブセットができたら、alphaという関数を使って、Cronbach αを出してみましょう。

1

alpha(a11, check.keys = T)

Reliability analysis
Call: alpha(x = a11, check.keys = T)

raw_alpha std.alpha G6(smc) average_r S/N ase mean sd median_r
0.69 0.69 0.61 0.42 2.2 0.008 3.4 0.75 0.45

Reliability if an item is dropped:

raw_alpha std.alpha G6(smc) average_r S/N ase mean sd median_r
sfmhpeac 0.49 0.49 0.33 0.33 0.97 0.0149 NA 0.33
sfmhengy 0.62 0.62 0.45 0.45 1.63 0.0111 NA 0.45
sfmhdprs- 0.67 0.67 0.50 0.50 1.99 0.0098 NA 0.50

以上のような結果が返ってきます（一部省略）。ここでは、たまにしたのような赤い警告が出てくる場合があります。

警告メッセージ:
alpha(a11, check.keys = T) で:
Some items were negatively correlated with total scale and were automatically reversed.
This is indicated by a negative sign for the variable name.

これは、逆転項目が逆転していないよという提示で、上の出力されたReliability if an item is droppedの部分をみてみれば、最後の変数名に**-**がついていることがわかります。これは、この変数が逆転していないよ〜というリマインドです。これができてから、変数の逆転作業を行いましょう。

ただし、pyschパッケージのすごい点として、逆転項目を逆転作業していなくても、Cronbach αを算出する際には、自動的に逆転してくれます。そのため、返ってきたCronbach αの数値は、信頼できるものとしても良いです。データを判断する時には、最初の返ってきた行をみてみましょう。

raw_alpha std.alpha G6(smc) average_r S/N ase mean sd median_r
0.69 0.69 0.61 0.42 2.2 0.008 3.4 0.75 0.45
ここの項目について、それぞれの意味は以下の通りです。
raw_alpha std.alpha G6(smc) average_r ase mean sd median_r
:–: :–: :–: :–: :–: :–: :–: :–: :–:
α係数 標準化されたα係数 ガットマンのラムダ6 項目間相関の平均値 αの標準誤差 平均 標準偏差 項目間相関の中位数
とりあえず、最初のraw_alphaをみて判断するのが良いでしょう。ここでは.69で、あまり高くはなかったですが、無理やり合成変数を作るというと作れるものでしょう。

また、この下にあるReliability if an item is droppedには、特定の変数を切り捨てる時に、α係数がどのように変化するのかを出しています。これを参照して、どのように合成変数を作成した方が良いかを判断する必要があります。

合成変数の作成にもいろんな方法がありますが、以下には一つ見本として出します。

1
2

jgss2012 <- jgss2012 %>% 
  mutate(mentalhealth = (sfmhpeac + sfmhengy + sfmhdprs)/3)

Beginner for Beginners 02: データと変数作成

Introduction

　前回では、パッケージの準備、データの読み込み、変数名の変更を行いました。本回では、引き続き、四則演算、変数の作成、合併について取り上げます。ここでは、すぐに使える、かつ私がいつも使っている手法を中心に紹介している予定ですが、ここで紹介されたもの以外にも、多様な手法があります。基本的にはtidyverseのパッケージによる処理方法をご紹介します。

データの準備

　では、前回で用意できたプロジェクトを開けて、libraryを読み込んでください。

1

library("tidyverse")

　プロジェクトには、jgss2012のデータが確実に入ってあることを確認しましょう。

1

view(jgss2012)

以上のコードを走らせることで、データセットの新しいウィンドが自動的に開かれ、データの中身について確認できます。なんとなくデータがちゃんと入ってあることを確認できたら、スクリプトに戻って、今回の内容をやってみましょう。
または、R Studioの右にあるデータ欄から、jgss2012をクリックして、データセットを確認することもできます。

四則演算

Rでは、もちろん四则运算ができます。数値による四则运算はもちろん、変数間の四则运算もできます。四则运算によって、新しい合成変数などを作成することができます。
例えば、dobyearという出生年度から年齢を算出したい場合は、

1

jgss2012$age <- 2011-jgss2012$dobyear

と打ちます。前回で説明したように、<-は付与する意味で、調査を行なった年2011から出生年度を引いた結果を、左のageという変数に書き込むということを指します。
演算したら、データがきちんと入っているかをチェックしましょう。

1
2

jgss2012$age %>% 
  str()

num [1:4667] 87 86 84 82 82 82 81 81 80 80 …

1
2

jgss2012$age %>% 
  head()

[1] 87 86 84 82 82 82

1
2

jgss2012$age %>% 
  table()

以上の三つの方法が少なくともデータがきちんと入っているかを確認することができる。以上のいずれかで確認したら良いと思います。

変数の作成

これから実際の分析を行う前には、欠損値などの前処理をする必要ができています。JGSSのような社会調査データでは、基本的に欠損値や「わからない」の答えを「999」などの数値で入力されるケースがほとんどである。この場合には、コードブックを参照しながら、データの前処理を行いましょう。

欠損値の処理

欠損値の処理には、多くの方法がありますが、ここでdyplrを使用する方法と、一般的な方法(?)の二種類を紹介します。
まず、標準関数を使用すると、以下のコードを使用します。

1
2

jgss2012$xjobhwk[jgss2012$xjobhwk == 888 
　　　　　　　　　| jgss2012$xjobhwk == 999] <- NA　# |はあるいはの意味

ここでは、888→わからない、999→無回答の二つの欠損値があるので、その二者を同時に処理すると、あるいはの意味で、|を入れます。このコードは、jgss2012のデータセットの中のxjobhwkという変数のなかの、888と999を欠損値NAとして処理してください、ということです。

また、dyplrを使用する場合、na_ifの関数を使用します。

1
2
3
4
5

jgss2012 <- jgss2012 %>% 
  mutate(xliveyr = na_if(xliveyr, 99))

jgss2012$xliveyr %>% 
  table()
1 2 3 4 5 6 7 8
774 111 253 204 454 800 668 1393

ここでは、99という欠損値が存在する変数xliveyrを、mutate関数で処理した。意味としては、jgss2012のデータセットの中のxliveyrという変数のなかの、全ての99を欠損値として変換してください。という意味です。

変数の変換

以上では、欠損値の処理方法について紹介しました。これからは、変数の変換とりわけ値の再付与について取り上げます。
JGSS2012の性別を例にします。性別では１→男性、２→女性として、データセットに入っていることをまず確認してみましょう。

1
2

jgss2012$sexa %>% 
  table()
1 2　
2148 2519

このような２値を、１と０に男性ダミー変数を作ると、以下のようなコードを書きます。

1
2
3

jgss2012 <- jgss2012 %>% 
  mutate(male_dummy = case_when(sexa == 2 ~ 0,
                                sexa == 1 ~ 1))

ここでも、dyplrパッケージの中のmutate関数を使用しています。今回は、na_ifという欠損値を処理する関数ではなく、case_whenという条件文を使用します。
ここでは、JGSS2012のデータセットのにあらためmale_dummyという変数を作って、sexaの変数の中の1を1のままとして、2を0に変換する意味です。また、この変換は、文字型でも使用できます。

1
2
3
4
5

jgss2012 <- jgss2012 %>% 
  mutate(gender = case_when(sexa == 1 ~ "男",
                            sexa == 2 ~ "女"))
jgss2012$gender %>% 
  table()
女 男
2519 2148

変数の作成

それから、データセットの中から、条件に相応しい項目だけを抽出して新しい変数を生成したりすることができます。
例えば、男性で、65歳以上のサンプルを抽出して、OBとして新しいダミー変数を作成したい場合は、以下のコマンドでできます。

1
2
3
4

jgss2012$ob <- NA
jgss2012$ob[jgss2012$male_dummy == 1 & jgss2012$ageb > 65] <- 1
jgss2012$ob %>% 
  table()

まず、OBという変数をあらかじめ作り、男性ダミーが１と年齢が65以上の条件にあうサンプルだけに１を付与する意味です。　　

それから、サンプルの中で、どれかに合えば良いケースもあるでしょう。その場合には、前節で取り上げた|を使用します。
例えば、ここで、学歴について、初等教育、中等教育、高等教育について分けたい場合は、以下のように、それぞれについて付与します。

1
2
3
4
5
6
7
8

jgss2012$educ <- NA
jgss2012$educ[jgss2012$xxlstsch == 1|jgss2012$xxlstsch == 2|
                jgss2012$xxlstsch == 3|jgss2012$xxlstsch == 8] <- "primary"
jgss2012$educ[jgss2012$xxlstsch == 4|jgss2012$xxlstsch == 5|
                jgss2012$xxlstsch == 6|jgss2012$xxlstsch == 9|
                jgss2012$xxlstsch == 10|jgss2012$xxlstsch == 11] <- "secondary"
jgss2012$educ[jgss2012$xxlstsch == 7|jgss2012$xxlstsch == 12| 　　　　　　　　　　jgss2012$xxlstsch ==13] <- "tertiary"

また、もっと簡潔な方法について、slackをご参照ください（次回追加）。 では、今日はここまで。

Beginner for Beginners 01: データと準備作業

Introduction

　このシリーズは、Rを最初からやり直そうとする筆者のメモと、一橋大学佐藤圭一（学部・院）ゼミの皆様のために、書いたものである。主には、筆者が疑問に思う点や、ググってもなかなかまとまった答えが出てこない部分に着目して書いたものである。単純にどうやってコードを書くよりも、筆者なりに理解・説明・解釈を加えて行く予定ではあるが、必ずしも正しいわけではないことに要注意。筆者の進捗によって、週一あるいは隔週一の頻度でまとまってメモるものになるでしょう。

RとR Studioの準備作業

　詳細のインストール等の準備作業とよくある問題に会った時の対処法について、麦山亮太さんが日本語による説明が非常に詳細に書かれているので、そちらをご参照ください。

パッケージについて

インストール

　分析上には、複数のパッケージを使用することとなるので、実際に使用する前に、それらをインストールする必要がある。パッケージをインストールする場合には、以下のコードが使用される。

1

install.packages("tidyverse")

tip

コマンドを実行するためのショートカットとして、
Windowsはcontrl+enterで、

Mac OSはcommand+enterです。

一つずつインストールすると、非常に面倒な作業なので、ここでは初心者の筆者がよく使用される（使ったことがある）パッケージを一斎にインストールすることができるように以下のコードを作成した。

1
2
3
4

useful.packages <- c("tidyverse", "broom", "sna",
                     "igraph","traminer","psych")
install.packages(useful.packages, 
                repos = "https://cran.rstudio.com")

これを、コピペして、ランしてください。少々時間がかかる可能性がある。

読み込む

作業を行う前には、毎回R Studioを立ち上げたあと、最初に行うべきことは、パッケージの読み込むと考えている。パッケージが読み込まれないと、コードを書いても反応しないことがあるので、その場合には、読み込むことが忘れているかをチェックしましょう。

1

library("tidyverse")

以上で準備作業が終わったので、次にデータをR Studioに導入してみよう。

データに関して

独自の調査を行う

実際の分析では、アンケート調査などで独自のデータを用意し、エクセル（一橋からアクセルあり）などを用いて、データの下処理を行うことが普通である。調査で得られた一人一人のデータ（個票データ）を、一つのファイルに統合し、R Studioに読み込む。

公開データを使用する

または、研究目的を達成するために、独自の調査を行うより、公開されている社会調査のデータを使用することはむしろ適切であるケースがあるでしょう。例えば、毎年行われている日本版総合社会調査(JGSS)、東京大学などで十年ごとに行われている、社会階層と社会移動に関する調査(SSM)などの調査データは、日本・世界中の複数のデータアーカイブに寄付され、特定の手順をへた後に、卒業論文や修士論文、博士論文などの使用されることが認められる。

データアーカイブ

以上のJGSSなどの公開データは、複数のデータアーカイブに寄付されている。例えば、日本国内における有名なデータアーカイブとして、東京大学社会科学研究所付属社会調査・データアーカイブ研究センターがあり、世界的には、アメリカのミシガン大学の政治・社会調査のための大学協会Inter-university Consortium for Political and Social Research, ICPSRがあり、ドイツのGESIS – Leibniz Institute for the Social Sciences、中国のChinese National Survey Data Archiveなどがある。
以上のデータアーカイブには、東大社研のデータにアクセスするために、事前に申請書を書かなければならないが、その他のアーカイブには、一橋のIDをログイン（ICPSR)か、アカウントを申請し、学術・教育にのみ使用される協議に同意することで、簡単に社会調査のデータをダウンロードすることができる。
このTutorialには、筆者個人の研究目的と、皆さんに実感を掴めるように、JGSS-2012のデータを使用する。データのダウンロードには、以上のICPSRあるいは、GESISで各自ダウンロードするようにしてください(検索結果のページに繋げてあるので、どれかをクリックしてください)。

Projectを作成する

関わっている作業が増えて行ったり、取っている授業数が多くなったりすることで、それぞれの作業と授業ごとにフォルダーに分けて管理することが（私なりに）普通でしょう。R Studioのプロジェクトもまさに同じようなイメージで掴めれば良いと思われる。それぞれのデータや、分析結果、作成した図などを、別々のフォルダーに管理することで、わかりやすくなる。
Projectの作成に関する説明は、ここでは省略するが、具体的にこのサイトをご参照。プロジェクトを作成できたら、以上でダウンローとしたデータをプロジェクトのフォルダーに保存しましょう。

データを読み込む

以上の準備作業が完了したら、ダウンロードできたデータをR Studioに導入しましょう。普段皆さんがアクセスすることが多いデータの形式として、.xlsxと.xlsのエクセルファイル、.csvのCSVファイル、まれに.staのSTATAファイル、.savのSPSSファイル、そして、ICSPRからデータをアクセスした場合、.rdaのR専用のデータ形式もあるかと思います。
エクセルでデータの書類を行った場合、.csvのCSVに変換して保存することがおすすめ。CSVファイルは、さまざまな統計ソフトや、プログラミング言語との相性がよく、エクセルファイルが起こりうる不具合も少ない。
では、それぞれのファイルをR Studioに導入しましょう。

CSVファイル

データの導入は、基本的に似たようなコマンドを実行します。CSVファイルの場合は、read.csvは、Rの標準コードで、read_csvはtidyverseを読み込んだ場合に使えるコード。

1
2

read.csv("JGSS-2012.csv") #ファイル名をそのままコピーし、.csvの拡張子も忘れずに。
read_csv("JGSS-2012.csv") #tidyverse的な書き方

このように、データを読み込んだだけでは、コンソール(console)でデータの状況に関する説明が表示されるだけなので、データをデータフレイムとして保存する必要がある。データフレイムを作るのに、個人の好みで、適当な名前をつけ、<-を使って、データフレイムとしてR Studioに認識させる。
